# Vercel: 프론트엔드 배포의 표준

## **vercel**은 웹 프로젝트, 특히 프론트엔드 애플리케이션을 빠르고 간편하게 배포하고 호스팅할 수 있도록 돕는 클라우드 플랫폼입니다. 인기 프레임워크인 **Next.js**를 만든 회사에서 운영하고 있습니다.

## 💡주요 특징

1. **자동 배포 (CI/CD)**

- GitHub, GitLab, Bitbucket 등 코드 저장소와 연동됩니다.
- 코드를 `git push`하는 순간, Vercel이 자동으로 빌드하여 실시간 서비스에 반영합니다.

2. **프리뷰 배포(Preview Deployments)**

- 새로운 기능을 개발할 때마다 고유한 Preview URL을 생성합니다.
- 실제 배포 전, 팀원들과 수정 사항을 미리 확인하고 리뷰하기에 최적화되어 있습니다.

3. **서버리스 및 엣지 기능**

- 서버를 직접 관리할 필요가 없는 Serverless Functions를 지원합니다.
- 전 세계 곳곳의 엣지 네트워크(Edge Network)를 사용해 사용자에게 가장 가까운 곳에서 데이터를 전달, 속도를 극대화합니다.

---

## 🛠️ Vercel 배포 워크플로우

1. **코드 작성 (Local)**: 내 컴퓨터에서 웹사이트 코드를 열심히 만듭니다.
2. **업로드 (Push)**: 작성한 코드를 GitHub 같은 저장소에 올립니다.
3. **감지 (Detect)**: Vercel은 GitHub에 새 코드가 올라온 것을 실시간으로 감지합니다.
4. **빌드 (Build)**: Vercel 서버에서 코드를 실제 서비스 가능한 상태로 조립합니다.
5. **배포 (Deploy)**: 조립이 끝나면 전 세계 서버로 뿌리고, 접속 가능한 URL을 생성합니다.
6. **확인 (Live)**: 사용자가 웹사이트에 접속합니다.
