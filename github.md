# GitHub

## GitHub란?

GitHub는 개발자들이 프로그램 소스 코드를 인터넷에 저장하고 관리할 수 있는 서비스이다.
코드의 변경 이력을 기록하는 **Git**을 기반으로 동작하며, 여러 사람이 하나의 프로젝트를 함께 개발할 때 협업을 쉽게 할 수 있도록 도와준다.

또한 다양한 오픈소스 프로젝트가 공개되어 있어 다른 개발자들의 코드를 참고하거나 학습하는 데에도 활용된다.

## GitHub의 주요 역할

1. 코드 버전 관리

GitHub에서는 Git을 이용하여 코드 변경 기록을 저장할 수 있다.
이를 통해 코드가 언제 수정되었는지 확인할 수 있고 문제가 생기면 이전 상태로 되돌릴 수 있다.

2. 협업 환경 제공

여러 개발자가 같은 프로젝트에서 작업할 때 코드 충돌을 줄이고 작업 내용을 공유할 수 있다.
또한 Pull Request 기능을 통해 코드 리뷰를 진행할 수도 있다.

3. 오픈소스 플랫폼

많은 개발자들이 자신의 프로젝트를 공개하여 다른 사람들과 지식을 공유한다.
이러한 공개 저장소를 통해 다양한 개발 방법과 기술을 배울 수 있다.

4. 개발 활동 기록

GitHub에는 개발자가 참여한 프로젝트와 활동 기록이 남기 때문에 개인 포트폴리오로도 활용된다.
기업에서는 개발자의 GitHub 활동을 참고하여 프로젝트 경험을 확인하기도 한다.

## Branch 사용 목적

Branch는 기존 코드에 영향을 주지 않고 새로운 작업을 진행하기 위한 독립적인 작업 공간이다. 개발자는 기본 브랜치를 유지한 상태에서 새로운 기능을 개발하거나 오류를 수정할 때 별도의 브랜치를 만들어 작업할 수 있다. 이렇게 하면 기존 코드에 영향을 주지 않고 안전하게 개발을 진행할 수 있으며, 작업이 완료된 후에는 해당 브랜치를 기존 브랜치에 병합하여 프로젝트에 반영할 수 있다.
Git 명령어 사용

Git에서는 코드의 변경 내용을 기록하고 관리하기 위해 여러 명령어를 사용한다. 주요 명령어는 다음과 같다.

- git add .
  변경된 파일을 스테이징 영역(Staging Area)에 추가하는 명령어이다.

- git commit -m "메시지"
  스테이징 영역에 추가된 변경 사항을 commit 단위로 기록하는 명령어이다. 이때 메시지를 작성하여 어떤 변경이 이루어졌는지 설명할 수 있다.

- git push origin (브랜치명)
  로컬 저장소에서 작업한 내용을 원격 저장소로 업로드하는 명령어이다.

- Git의 변경 기록 관리

Git은 프로젝트의 모든 변경 내용을 commit 기록으로 저장한다. 이러한 기록을 통해 어떤 파일이 언제 수정되었는지 확인할 수 있으며, 필요할 경우 이전 버전의 코드 상태로 되돌리는 것도 가능하다. 이 기능은 프로젝트를 안정적으로 관리하는 데 중요한 역할을 한다.

- Git을 활용한 협업

Git을 사용하면 여러 개발자가 동시에 하나의 프로젝트를 작업할 수 있다. 각 개발자는 자신의 작업을 별도의 브랜치에서 진행하고, 작업이 완료되면 브랜치를 병합하여 프로젝트에 반영할 수 있다. 이를 통해 코드 충돌을 줄이고 체계적으로 프로젝트를 관리할 수 있다.

## Pull Request(PR)

Pull Request는 내가 작업한 브랜치를 다른 브랜치에 병합해 달라고 요청하는 기능이다.
이 과정을 통해 다른 개발자들이 코드를 확인하고 의견을 남길 수 있다.

**Pull Request 생성 과정**

1. GitHub 저장소 페이지로 이동

2. Compare & pull request 버튼 선택

3. 병합할 브랜치(base)와 작업 브랜치(compare) 확인

4. 제목과 설명을 작성한 후 Create pull request 클릭

## 코드 리뷰 반영

Pull Request가 생성되면 팀원들이 코드를 확인하고 피드백을 남길 수 있다.

- **리뷰 확인**

PR 페이지의 Files changed 탭에서 수정된 코드와 리뷰 내용을 확인할 수 있다.

- **수정 반영 과정**

코드를 수정한 뒤 다시 아래 명령어를 실행하면 기존 PR에 자동으로 업데이트된다.

git add .

git commit -m "수정 내용"

git push origin 브랜치명

리뷰어가 승인을 하면 Merge pull request 버튼을 통해 메인 브랜치에 병합할 수 있다.

## Issue 기능

Issue는 프로젝트에서 해야 할 작업이나 발생한 문제를 기록하고 관리하는 기능이다.

- **Issue 생성 방법**

  1.저장소 상단의 Issues 메뉴 선택

  2.New issue 클릭

  3.제목과 내용을 작성 후 등록

또한 담당자(Assignees)를 지정하거나 라벨(Labels)을 설정하여 작업을 체계적으로 관리할 수 있다.
