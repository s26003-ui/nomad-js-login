## fetch API의 정의와 탄생 배경
fetch는 자바스크립트에서 브라우저가 서버와 데이터를 주고받을 수 있도록 지원하는 내장 비동기 함수입니다.

과거에는 XMLHttpRequest라는 복잡하고 다루기 힘든 방식을 사용했으나, 이를 대체하기 위해 등장했습니다. fetch는 Promise 객체를 기반으로 설계되어 비동기 작업을 훨씬 직관적이고 깔끔하게 처리할 수 있게 해줍니다.

## 왜 비동기가 중요한가?
웹 페이지에서 서버에 데이터를 요청할 때 응답이 오기까지는 시간이 걸립니다. 만약 이 작업을 **동기(Synchronous)**적으로 처리한다면, 데이터가 올 때까지 웹 페이지는 아무런 동작도 할 수 없는 '프리징(Freezing)' 상태가 됩니다.

fetch는 비동기 방식을 사용하기 때문에:

서버에 요청을 던져두고, 브라우저는 다른 작업을 계속 수행합니다.

응답이 도착하면 그때 비로소 준비된 로직을 실행합니다.

덕분에 사용자는 끊김 없는 웹 경험을 할 수 있습니다.

## fetch의 응답 메커니즘
fetch를 통해 데이터를 가져올 때는 물리적인 통신 절차상 두 번의 기다림이 발생합니다.

Response 객체 수신 (1단계): 서버로부터 "연결 성공(200 OK)" 혹은 "페이지 없음(404)" 같은 응답 헤더 정보를 먼저 받습니다. 이 단계에서는 실제 데이터 본문이 도착하기 전이므로 본문의 내용을 바로 볼 수는 없습니다.

Body 데이터 변환 (2단계): 서버가 보낸 실제 데이터(JSON, 텍스트, 이미지 등)를 읽어내는 단계입니다. 서버는 보통 데이터를 잘게 쪼개서 보내기 때문에, 이를 다시 우리가 쓸 수 있는 형태로 조립하는 과정(.json(), .text() 등)이 필요하며 이 역시 비동기로 이루어집니다.

## HTTP 요청 방식(Methods)의 개념
fetch는 단순히 정보를 가져오는 기능 외에도 다양한 목적의 통신을 수행합니다. 이를 HTTP Methods라고 합니다.

GET (조회): 서버의 자원을 가져올 때 사용합니다. (게시글 읽기, 검색 결과 등)

POST (생성): 서버에 새로운 데이터를 저장해달라고 요청할 때 사용합니다. (회원가입, 글쓰기 등)

PUT/PATCH (수정): 이미 서버에 있는 데이터를 고칠 때 사용합니다. (회원정보 수정 등)

DELETE (삭제): 서버의 데이터를 지울 때 사용합니다. (댓글 삭제 등)

## 비동기를 처리하는 두 가지 문법 스타일
fetch가 반환하는 Promise를 다루는 방법에는 두 가지가 있습니다.

- .then() 방식 (기존 방식)
"작업이 성공하면 그다음에(then) 이걸 하고, 또 성공하면 그다음에 저걸 해"라고 명령을 연결하는 방식입니다. 체이닝(Chaining) 구조라고도 하며, 로직이 길어지면 가독성이 떨어질 수 있습니다.

- async / await 방식 (실무 권장 방식)
"이 작업이 끝날 때까지 여기서 잠시 기다려(await)"라고 명시하는 방식입니다. 비동기 코드를 마치 일반적인 동기 코드처럼 위에서 아래로 읽히게 만들어주어 가독성이 비약적으로 상승합니다. 현대 개발 환경에서 가장 선호되는 방식입니다.

## fetch 사용 시의 필수 주의사항
- 에러 처리의 특이점
fetch는 네트워크 자체가 끊기지 않는 이상, 서버에서 **404(Not Found)**나 500(Internal Server Error) 응답을 받아도 에러를 발생시키지 않습니다(reject 되지 않음).
따라서 개발자는 반드시 코드 내에서 response.ok 속성을 확인하여 "진짜 성공적인 응답인가?"를 직접 체크해야 합니다.

- 데이터의 문자열화 (Serialization)
서버로 데이터를 보낼 때(POST 등), 자바스크립트의 객체(Object) 형태 그대로 보내면 서버는 이해하지 못합니다. 반드시 JSON.stringify()를 사용하여 데이터를 문자열로 변환(직렬화)하는 포장 과정이 필요합니다.
