# Git

---

## Git란?

**Git**은 파일이나 프로그램 코드의 변경 기록을 관리하는 분산 버전 관리 시스템이다. Git은 프로젝트에서 파일이 언제, 어떻게 수정되었는지 기록하여 이전 버전으로 되돌리거나 변경 내용을 확인할 수 있게 해준다. 또한 여러 사람이 동시에 같은 프로젝트를 작업할 수 있도록 협업 기능을 제공하며, 각 개발자의 컴퓨터에도 전체 기록이 저장되는 분산 구조를 가지고 있다. Git은 보통 GitHub, GitLab 같은 온라인 저장소 서비스와 함께 사용되어 코드 관리와 협업을 효율적으로 할 수 있게 해준다.

## Git의 주요 기능

**Git**은 프로그램 코드나 파일의 변경 기록을 관리하는 분산 버전 관리 시스템(Distributed Version Control System)이다. Git을 사용하면 프로젝트의 수정 이력을 체계적으로 관리할 수 있으며, 여러 개발자가 동시에 하나의 프로젝트를 작업할 때 효율적으로 협업할 수 있다.

1. 버전 관리

Git은 파일이나 코드가 수정될 때마다 그 내용을 기록하여 관리한다. 이를 통해 코드가 언제, 어떻게 변경되었는지 확인할 수 있으며 문제가 발생했을 경우 이전 버전으로 쉽게 되돌릴 수 있다.

2. 협업 기능

Git은 여러 명의 개발자가 동시에 하나의 프로젝트를 작업할 수 있도록 지원한다. 각 개발자는 자신의 작업 내용을 기록하고 나중에 다른 개발자의 작업과 병합할 수 있어 협업 과정에서 발생할 수 있는 충돌을 줄일 수 있다.

3. 변경 기록 관리

Git에서는 파일의 변경 내용을 commit이라는 단위로 저장한다. Commit을 통해 프로젝트의 수정 이력을 확인할 수 있으며 특정 시점의 코드 상태로 복구하는 것도 가능하다.

4. 원격 저장소 활용

Git은 로컬 저장소뿐만 아니라 원격 저장소와 연결하여 코드를 관리할 수 있다. 예를 들어 GitHub 같은 서비스를 이용하면 프로젝트를 온라인에 저장하고 다른 개발자들과 코드를 공유하며 협업할 수 있다.

## Git 명령어

- git init
  : Git 저장소를 초기화하는 명령어입니다. 이 명령어를 사용하면 기존 디렉토리를 Git 저장소로 변환하거나, 새로운 프로젝트를 시작할 때 Git 저장소를 생성할 수 있습니다.
- git config
  : Git의 설정을 관리하는데 사용되는 명령어입니다. Git은 사용자의 이름, 이메일 주소, 에디터 설정 등을 저장하고 이를 사용하여 커밋 메시지 작성 등에 활용합니다.
- git status
  : 현재 작업 디렉토리와 인덱스의 상태를 보여주는 명령어입니다. 이 명령어를 실행하면 현재 프로젝트에서 어떤 파일들이 수정되었고, 어떤 파일들이 인덱스에 추가되었는지, 커밋이 필요한 변경 사항이 있는지 등을 확인할 수 있습니다.
- git add
  : Git에서 변경된 파일들을 스테이지 영역에 추가하는 명령어입니다.
- git commit
  : 스테이지 영역에 추가된 변경 사항들을 로컬 저장소에 커밋하는 명령이입니다.
- git log
  : 현재 브랜치의 커밋 이력을 보여주는 명령어입니다. 이 명령어를 사용하면 저장소의 커밋 기록을 확인할 수 있습니다.
- git tag
  : Git 저장소에서 태그를 관리하는데 사용되는 명령어입니다. 태그는 특정 시점의 커밋을 의미하는 이름으로 붙이는 것을, 주로 소프트웨어 버전을 관리하거나 중요한 이벤트를 표시하는데 사용됩니다.
- git diff
  :코드의 변경사항을 확인할 수 있어 소스 코드의 상태를 비교하고 이해하는데 도움이 됩니다.
- git reset
  : Git에서 사용되는 명령어로서, 이전 커밋들과의 관계를 재설정하는 역할을 합니다.이전 커밋들로 돌아가거나, Staging Area에 있는 파일들을 변경하는 등의 작업을 수행할 수 있습니다.
- git revert
  : 이전 커밋을 취소한 새로운 커밋을 생성하는 Git 명령어입니다. 기존의 커밋을 변경하지 않고 변경 사항을 되돌리는 방법으로 사용됩니다.
- git stash
- git branch
- git checkout
  : Git에서 브랜치를 전환하거나 작업 디렉토리의 파일을 변경하는데 사용되는 명령어입니다.
  -git rebase
  : 주로 브랜치를 통합하거나 커밋들을 재배치하는데 사용되는 Git 명령어입니다.현재 브랜치의 커밋들을 다른 브랜치의 커밋들 위로 이동시킬 수 있습니다.
- git clone
  : 원격 저장소에서 코드를 가져와서 로컬 환경에 복제하는 Git 명령어입니다. 기본적으로 원격 저장소의 코드를 로컬로 복제하여 새로운 Git 저장소를 생성합니다.
- git remote
  : 로컬 Git 저장소에 등록된 원격 저장소를 관리하는 데 사용되는 명령어입니다. 원격 저장소는 코드를 저장하고 다른 개발자와 협업하는 데 사용되며, 주로 GitHub, GitLab, Bitbucket 등의 원격 저장소 호스팅 서비스와 연결됩니다.
- git push
  : 로컬 저장소의 변경 사항을 원격 저장소에 업로드하는 Git명령어입니다. 로컬 저장소에서 작업한 내용을 원격 저장소로 올리는데 사용됩니다.
- git fetch
  : 원격 저장소의 최신 정보를 로컬 저장소로 가져오는 Git 명령어입니다.
- git pull
  : 원격 저장소에서 변경된 내용을 로컬 저장소로 가져오는 작업을 수행하는 Git명령어입니다.
