# Git
---
## Git란? 
**Git**은 파일이나 프로그램 코드의 변경 기록을 관리하는 분산 버전 관리 시스템이다. Git은 프로젝트에서 파일이 언제, 어떻게 수정되었는지 기록하여 이전 버전으로 되돌리거나 변경 내용을 확인할 수 있게 해준다. 또한 여러 사람이 동시에 같은 프로젝트를 작업할 수 있도록 협업 기능을 제공하며, 각 개발자의 컴퓨터에도 전체 기록이 저장되는 분산 구조를 가지고 있다. Git은 보통 GitHub, GitLab 같은 온라인 저장소 서비스와 함께 사용되어 코드 관리와 협업을 효율적으로 할 수 있게 해준다.
## Git의 주요 기능
**Git**은 프로그램 코드나 파일의 변경 기록을 관리하는 분산 버전 관리 시스템(Distributed Version Control System)이다. Git을 사용하면 프로젝트의 수정 이력을 체계적으로 관리할 수 있으며, 여러 개발자가 동시에 하나의 프로젝트를 작업할 때 효율적으로 협업할 수 있다.
1. 버전 관리

Git은 파일이나 코드가 수정될 때마다 그 내용을 기록하여 관리한다. 이를 통해 코드가 언제, 어떻게 변경되었는지 확인할 수 있으며 문제가 발생했을 경우 이전 버전으로 쉽게 되돌릴 수 있다.

2. 협업 기능

Git은 여러 명의 개발자가 동시에 하나의 프로젝트를 작업할 수 있도록 지원한다. 각 개발자는 자신의 작업 내용을 기록하고 나중에 다른 개발자의 작업과 병합할 수 있어 협업 과정에서 발생할 수 있는 충돌을 줄일 수 있다.

3. Branch 기능

Branch는 기존 코드에 영향을 주지 않고 새로운 작업을 진행하기 위한 독립적인 작업 공간이다. 개발자는 기본 브랜치를 유지한 상태에서 새로운 기능을 개발하거나 오류를 수정할 때 별도의 브랜치를 만들어 작업할 수 있다. 작업이 완료되면 해당 브랜치를 기존 브랜치와 병합하여 프로젝트에 반영할 수 있다.

4. 변경 기록 관리

Git에서는 파일의 변경 내용을 commit이라는 단위로 저장한다. Commit을 통해 프로젝트의 수정 이력을 확인할 수 있으며 특정 시점의 코드 상태로 복구하는 것도 가능하다.

5. 원격 저장소 활용

Git은 로컬 저장소뿐만 아니라 원격 저장소와 연결하여 코드를 관리할 수 있다. 예를 들어 GitHub 같은 서비스를 이용하면 프로젝트를 온라인에 저장하고 다른 개발자들과 코드를 공유하며 협업할 수 있다.
## Branch 사용하기

Branch는 기존 코드에 영향을 주지 않고 새로운 작업을 진행하기 위한 독립적인 작업 공간이다. 개발자는 기본 브랜치를 유지한 상태에서 새로운 기능을 개발하거나 오류를 수정할 때 별도의 브랜치를 만들어 작업할 수 있다. 이렇게 하면 기존 코드에 영향을 주지 않고 안전하게 개발을 진행할 수 있으며, 작업이 완료된 후에는 해당 브랜치를 기존 브랜치에 병합하여 프로젝트에 반영할 수 있다.
Git 명령어 사용

Git에서는 코드의 변경 내용을 기록하고 관리하기 위해 여러 명령어를 사용한다. 주요 명령어는 다음과 같다.

git add .
변경된 파일을 스테이징 영역(Staging Area)에 추가하는 명령어이다.

git commit -m "메시지"
스테이징 영역에 추가된 변경 사항을 commit 단위로 기록하는 명령어이다. 이때 메시지를 작성하여 어떤 변경이 이루어졌는지 설명할 수 있다.

git push origin (브랜치명)
로컬 저장소에서 작업한 내용을 원격 저장소로 업로드하는 명령어이다.

Git의 변경 기록 관리

Git은 프로젝트의 모든 변경 내용을 commit 기록으로 저장한다. 이러한 기록을 통해 어떤 파일이 언제 수정되었는지 확인할 수 있으며, 필요할 경우 이전 버전의 코드 상태로 되돌리는 것도 가능하다. 이 기능은 프로젝트를 안정적으로 관리하는 데 중요한 역할을 한다.

Git을 활용한 협업

Git을 사용하면 여러 개발자가 동시에 하나의 프로젝트를 작업할 수 있다. 각 개발자는 자신의 작업을 별도의 브랜치에서 진행하고, 작업이 완료되면 브랜치를 병합하여 프로젝트에 반영할 수 있다. 이를 통해 코드 충돌을 줄이고 체계적으로 프로젝트를 관리할 수 있다.
